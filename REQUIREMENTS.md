# AI Text Polisher (Typless-like App) 要件定義書 v2

## 1. プロジェクト概要
音声入力（マイク）やラフなメモ書きに含まれる「フィラー（えー、あの等）」や「言い直し・訂正」を、
ローカルSTT（whisper.cpp）＋ローカルLLM（Ollama）を用いてリアルタイムに整形・ビジネス文章化するデスクトップアプリケーション。

**処理フロー:**
```
[ショートカット押下] → [マイク録音] → [whisper.cpp でSTT] → [Ollama で整形] → [クリップボード貼り付け]
または
[ショートカット押下] → [選択テキスト取得] → [Ollama で整形] → [クリップボード貼り付け]
```

## 2. ターゲット環境
- **OS:** macOS (Apple Silicon / M1 Mac 以降)
- **メモリ:** 8GB RAM (最小構成)
- **STTエンジン:** whisper.cpp（CoreML/Metal加速） via `pywhispercpp`
- **LLMエンジン:** Ollama (ローカルAPIサーバーとして利用)
- **推奨LLMモデル:** `gemma3:4b`（速度と精度のベストバランス）
  - 予備モデル: `gemma2:2b`（超高速化が必要な場合）
- **推奨Whisperモデル:** `base` または `small`（日本語精度と速度のバランス）

## 3. アプリケーション機能要件

### 3.1. コア機能
1. **音声入力モード（Push-to-Talk）:**
   - ショートカットキー（例: `Cmd + Shift + K`）を**押している間**マイク録音。
   - 離したタイミングでSTT処理（whisper.cpp）を実行。
   - STT結果をOllamaで整形し、クリップボードに書き戻す。
   - 可能であれば自動的に `Cmd + V` を実行して貼り付ける。

2. **テキスト整形モード:**
   - 同ショートカットのワンタップ（短押し）で、現在選択中のテキストまたはクリップボードのテキストを取得。
   - OllamaAPIで整形し、クリップボードに書き戻す。

3. **グローバルショートカット:**
   - バックグラウンド動作時でもキャプチャできるグローバルホットキー（`pynput` 使用）。

### 3.2. UX/UI 要件
- **常駐方式:** メニューバーアプリとして常駐（Dockに表示しない）。
- **動作速度:** 1リクエストにつき 10秒以内（M1 Mac 8GB環境）。
  - STT（whisper.cpp Metal加速）: 2〜3秒
  - LLM整形（Ollama gemma3:4b）: 3〜7秒
- **インジケーター:** 処理中はメニューバーアイコンをアニメーション（スピナー）表示。
- **通知:** 完了時にmacOS通知を表示（整形後テキストのプレビューを含む）。
- **モード選択（メニューバーから切り替え）:**
  - **Standard:** 自然なビジネス文章に整形。
  - **Summarize:** 決定事項のみを抽出（言い直しの最後の結論のみ残す）。
  - **Bullet:** 箇条書き形式に整形。
  - **Email:** メール文面として整形。

### 3.3. macOS権限要件
- **マイク権限:** 音声入力のために必要。初回起動時にシステムダイアログで要求。
- **アクセシビリティ権限:** グローバルショートカットの検知および自動ペーストに必要。システム環境設定から手動で付与が必要。
- **アプリ起動時:** 権限が不足している場合、わかりやすいガイダンスを表示。

### 3.4. エラーハンドリング
- **Ollama未起動:** 「Ollamaが起動していません。ターミナルで `ollama serve` を実行してください。」と通知。
- **モデル未取得:** 「モデルが見つかりません。`ollama pull gemma3:4b` を実行してください。」と通知。
- **マイク権限なし:** システム環境設定へのリンク付きでガイダンス表示。
- **STT失敗:** フォールバックとしてクリップボードのテキストを整形対象とする。

## 4. プロンプト要件定義（核心部）

### 4.1. Standard モード（デフォルト）
```text
あなたは最高峰のリアルタイム・エディターです。
以下の【厳格なルール】に従って、入力された崩れた文章を完璧に整形してください。

【厳格なルール】
1. フィラー除去: 「えー、あの、その、えっと、あー」等の意味を持たない言葉はすべて削除。
2. 訂正の優先: 「A、いやB、やっぱりC」という言い直しがある場合、最後の「C」のみを真実として採用し、AとBは完全に無視すること。
3. 重複の統合: 同じ内容の繰り返し（例：「資料の、資料の件で」）は1つにまとめる。
4. 自然な敬語: 相手に失礼のない、かつ簡潔なビジネス敬語（です・ます調）で構成。
5. 出力制限: 解説、挨拶、補足説明（「以下は整形後です」等）は一切禁止。整形後の本文のみを出力すること。
```

### 4.2. Summarize モード
```text
あなたは議事録の要約専門家です。
入力された文章から「最終的な決定事項・結論のみ」を抽出してください。

【厳格なルール】
1. 言い直しや検討過程（「17日→18日→19日」）は最後の結論（「19日」）のみを残す。
2. フィラーや繰り返しはすべて除去。
3. 簡潔な体言止めまたはです・ます調で出力。
4. 出力は整形後の本文のみ。解説・補足は一切禁止。
```

### 4.3. Bullet モード
```text
入力された文章を、簡潔な日本語の箇条書きリストに変換してください。
各項目は「・」で始め、重複や言い直しは除去すること。
出力は箇条書きのみ。解説・補足は一切禁止。
```

### 4.4. Email モード
```text
入力された内容を、ビジネスメールの本文として整形してください。
適切な敬語を使用し、冒頭の挨拶（「お世話になっております。」）と末尾の締め（「よろしくお願いいたします。」）を付けること。
フィラー・言い直しは除去。出力はメール本文のみ。
```

## 5. 技術スタック

| レイヤー | 採用技術 | 理由 |
|---------|---------|------|
| GUI / メニューバー | **rumps** (Python) | macOSメニューバーアプリに特化、軽量 |
| グローバルショートカット | **pynput** | バックグラウンドキー検知 |
| 音声録音 | **sounddevice** + **numpy** | シンプルなPython音声入力 |
| STT | **pywhispercpp** (whisper.cpp) | M1 Metal加速、ローカル、高精度 |
| LLM整形 | **Ollama REST API** | ローカルLLM、プライバシー保護 |
| クリップボード | **pyperclip** | クロスプラットフォーム対応 |
| 自動ペースト | **pynput** (Cmd+V送信) | アクセシビリティ権限で動作 |

## 6. パフォーマンス目標
- **モデルロード時間（Ollama）:** 3秒以内（初回以降、keep_alive: -1 でメモリ常駐）
- **STT処理時間（whisper.cpp Metal加速）:**
  - 5秒以内の発話: 1〜2秒
  - 30秒の発話: 3〜5秒
- **LLM整形時間:**
  - 短文（50文字以下）: 3〜5秒
  - 長文（200文字以上）: 7〜10秒
- **合計（音声入力→貼り付け）:** 15秒以内を目標

## 7. セットアップ要件（ユーザー側）

```bash
brew install ollama

# 2. Python依存関係

# 3. whisper.cppモデルのダウンロード（初回のみ）
# アプリ起動時に自動ダウンロード（base モデル、約150MB）
```
